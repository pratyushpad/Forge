# Forge demo (Next.js / TypeScript)

A thin, clickable showcase: type or pick a math problem and watch the **base
model** and the **GRPO-tuned model** solve it side by side, reasoning trace
streaming live, then a results panel with the pass@1 bar and the reward curve.

The demo showcases the model — it is not the project. The substance is the
training/eval/quantize/serve pipeline in the repo root.

## How it works

- **Default (always works):** replays **real cached outputs** from both models
  (`public/examples.json`, generated by `python -m demo.gen_examples` against the
  actual base + GRPO-tuned models via Ollama) with a typewriter reveal. This is why
  the public page works even when the training box is off — the outputs are genuine,
  not fabricated.
- **Live mode (optional):** `app/api/generate/route.ts` streams from any
  OpenAI-compatible endpoint set via `FORGE_FALLBACK_URL` (+ `FORGE_FALLBACK_KEY`,
  `FORGE_FALLBACK_MODEL`) — e.g. the local vLLM/Ollama servers from `serve/`, or a
  hosted fallback. With nothing configured it returns 503 and the page uses cached
  outputs.

## Run locally

```bash
cd demo
npm install
npm run dev        # http://localhost:3000
```

Regenerate the cached outputs (needs Ollama serving base + forge-q4):
```bash
python -m demo.gen_examples
```

## Deploy to Vercel

The Next.js root is this `demo/` directory.

- **CLI:** `cd demo && npx vercel --prod` (first run links/creates the project).
- **Dashboard:** import the GitHub repo, set **Root Directory = `demo`**, deploy.

For live inference in production, set the `FORGE_FALLBACK_URL` env var in the Vercel
project to a reachable OpenAI-compatible endpoint. Without it, the deployed page runs
purely on the cached real outputs.
